HTML

  • The head element contains info that helps the browser to render the page correctly, they are not visible to the user.
  • The body element has the content visible to the user.
  • ul is an unordered list.
  • ol is an ordered list.
  • li are the list items.

CSS

  • 3 ways to use CSS: Inline, Internal and External.
  • Inline: directly inside the HTML file.
  • Internal: contain CSS rules for the webpage in the head section of the HTML file. Rules apply to that page.
  • External: Best. Keep all CSS rules in a seperate file.

Git

  • The git command must preceed every Git statement.
  • The checkout command is to move the working branch to a new branch.
  • The -b flag creates a new branch.
  • The add command adds modifications in the current working branch to the staging area.
  • The -A flag indicates to add all changes.
  • git pull commit -m "add starter code" is a message to describe what the commit contains.
  • git pull origin main is to check if the local branch is in sync with the base branch in GitHub.
  • git push origin "new branch" is to push changes to the remote branch.

JavaScript

  • JavaScript runs code from the first statement at the top of the page, down to the last statement at the bottom of the page.
  • A variable is a container that allows someone to store data in the code.
  • An Array is a single variable that is used to hold a group of data.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function is a reusable block of code that contains instructions to perform a specific task.
  • CODE ORDER IS VERY IMPORTANT!!!
